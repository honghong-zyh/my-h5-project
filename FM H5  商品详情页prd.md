# FM H5 商品详情页 PRD

## 产品概述

### 名称
**FastMoss H5 - 商品列表 & 商品详情页**

### 产品概念
- **移动端数据看板**：在 H5 内快速浏览 TikTok 选品/商品经营关键指标
- **从列表到详情的深度分析链路**：商品卡片 -> 详情页 -> 关联内容/达人/直播等子页面
- **可筛选、可排序、可分享导出**：围绕“发现-判断-留存/分享”闭环提升效率

### 产品愿景
为投放/运营/选品人员提供一个移动端即可随时查看商品表现、洞察来源内容/达人/直播构成，并能快速完成筛选排序、截图分享与导出线索的轻量工具。

### 目标
- 提供“商品列表 -> 商品详情 -> 子页面”完整浏览链路
- 支持关键指标的筛选、排序与查看示例（会员权限蒙层）
- 提供分享、导出等协作能力（至少 UI 与交互闭环）

## 目标用户

### 主要用户画像：跨境电商运营（25-35岁）
- 高频使用移动端查看数据（通勤/外出）
- 需要快速判断某商品是否值得跟进（销量、销售额、佣金、类目、国家等）
- 需要定位增长来源（关联视频/直播/达人）
- 有协作诉求：分享给团队、导出线索

## 成功指标（KPI）

### 主要 KPI
- **商品详情页到达率**：从列表进入详情的点击率 ≥ 15%（上线 4 周）
- **子页面打开率**：详情页中任一子页面（全部达人/关联直播/关联视频/评论等）打开率 ≥ 20%（上线 4 周）

### 次要 KPI
- **分享触发率**：详情页/子页面分享按钮点击率 ≥ 5%（上线 4 周）
- **导出触发率**：导出弹窗打开率 ≥ 3%（上线 4 周）

## 功能需求

### 1. 商品列表页（对应截图：商品列表）

**用户故事**：
作为运营，我想在商品列表中快速浏览和筛选商品，以便挑选值得进一步分析的商品。

**功能点**：
- 顶部导航：国家切换、登录入口（如有）
- 顶部搜索框：支持搜索商品/达人/店铺（至少 UI + 输入态）
- 筛选与排序：类目筛选、指标排序（例如“直播总销量排序”等）
- 商品卡片：
  - 商品图片、商品名称、价格、佣金比例
  - 国家/类目等标签
  - 核心指标展示：销售额、销量
  - 收藏/星标（UI 与状态）
- 列表滚动加载（MVP 可先用分页/模拟数据）
- 点击商品卡片进入商品详情页

**受理标准**：
- [ ] 列表页可正常滚动展示商品卡片
- [ ] 点击任一卡片进入对应商品详情页（带 productId）
- [ ] 搜索/筛选/排序的交互状态可见（高亮/选中态明确）

**优先级**：P0

---

### 2. 商品详情页（对应截图：FastMoss-商品详情）

**用户故事**：
作为运营，我想查看该商品的关键表现与来源构成，以便判断是否跟进投放/选品。

**页面结构（信息分区）**：
1) 顶部 Header
- 返回上一页
- 页面标题：FastMoss·商品详情

2) 商品信息卡（核心）
- 商品图、商品标题（支持省略）
- 价格（含划线价可选）
- 国家、类目标签
- 佣金比例、上架时间、体验分、总销量
- 操作按钮：SKU详情入口
- 操作 icon：刷新、收藏

3) 吸顶面板（滚动后出现）
- Tabs：趋势分析 / 成交分析 / SKU分析 / 达人分析 / 关联视频 / 关联直播 / 评论 / 相似品
- 时间范围切换：7天 / 28天 / 90天

4) 内容模块（按 Tab 对应 section 锚点）
- 趋势分析
- 成交分析
- SKU 分析
- 达人分析
- 关联视频
- 关联直播
- 评论
- 相似品

**受理标准**：
- [ ] 返回按钮可回到列表页
- [ ] 商品信息卡可展示上述字段
- [ ] 吸顶面板在满足条件时出现且不遮挡主要交互
- [ ] Tab 点击可滚动到对应模块

**优先级**：P0

---

### 3. 趋势分析模块（详情页内）

**用户故事**：
作为运营，我想查看销量/销售额等趋势，以便判断商品近期走势。

**功能点**：
- 指标卡：销量、销售额、带货达人、带货视频、带货直播、售价（可多选但限制最多 2 项）
- 折线图：展示选择的指标
- 会员权限蒙层：
  - 文案提示“当前数据仅限个人版以上会员查看…”
  - 按钮：升级会员、查看示例

**受理标准**：
- [ ] 指标卡支持选择/取消选择（最多 2 项）
- [ ] 蒙层出现时按钮可点击；点击“查看示例”隐藏蒙层

**优先级**：P1

---

### 4. 成交分析模块（详情页内）

**用户故事**：
作为运营，我想了解成交渠道/内容/投放占比，以便优化投放与内容策略。

**功能点**：
- 下拉切换：成交渠道占比 / 成交内容占比 / 成交投放占比
- 图表（饼图/条形等，MVP 可先用静态数据）

**受理标准**：
- [ ] 下拉可切换展示维度
- [ ] 对应图表随维度切换更新

**优先级**：P1

---

### 5. SKU 分析模块（详情页内）

**用户故事**：
作为运营，我想查看不同 SKU 在成交/库存维度的占比与贡献，以便调整商品结构。

**功能点**：
- SKU 维度选择：颜色/尺寸/规格等
- 指标选择：销量/销售额
- 展示：列表 + 占比（MVP 可用静态）

**受理标准**：
- [ ] 切换维度与指标后展示同步更新

**优先级**：P1

---

### 6. 达人分析模块（详情页内）

**用户故事**：
作为运营，我想看带货达人构成与头部达人表现，以便快速判断达人合作方向。

**功能点**：
- 近7天销量 Top3 达人榜单（领奖台样式）
- 入口：全部带货达人（打开全屏子页面）

**受理标准**：
- [ ] Top3 达人榜单可展示达人头像/昵称/核心数值
- [ ] 点击“全部带货达人”进入全屏列表

**优先级**：P0

---

### 7. 全部带货达人子页面（全屏弹窗/内置页）

**用户故事**：
作为运营，我想在全量达人列表中筛选类别并按不同指标排序，以便找到优质达人。

**页面结构**：
- 顶部 Header：返回、标题“全部带货达人”、分享、导出
- 顶部筛选条：
  - 左侧：类别筛选下拉（全部 / 视频达人 / 直播达人）
  - 右侧：排序控件（排序：总销量/销售额/粉丝数 + 升降序 + ArrowUpDown）
- 列表：达人卡片（头像、昵称、类目、粉丝、总销量、销售额等）

**受理标准**：
- [ ] 类别筛选下拉可打开/选择/关闭
- [ ] 排序下拉可选择指标；升降序切换生效
- [ ] 列表可滚动

**优先级**：P0

---

### 8. 关联视频模块（详情页内） + 全屏列表

**用户故事**：
作为运营，我想查看与商品相关的视频内容，并按播放/销量等排序，以便定位增长内容。

**功能点（全屏列表优先）**：
- 顶部：返回、标题“关联视频”、分享、导出
- 筛选：投流/未投流切换（如果有）
- 排序：排序下拉（发布时间/播放量/销量等） + 升降序按钮
- 列表：视频封面、标题、作者、国家、核心指标

**受理标准**：
- [ ] 排序控件交互一致：排序下拉 + 最右切换升降序
- [ ] 列表按当前排序字段与方向排序

**优先级**：P0

---

### 9. 关联直播模块（详情页内） + 全屏列表

**用户故事**：
作为运营，我想查看与商品相关的直播，并按直播总销量/粉丝数/累计观看人数排序，以便发现优质直播间。

**功能点（全屏列表）**：
- 顶部：返回、标题“关联直播”、分享、导出
- 筛选/排序（严格与参考交互一致）：
  - 右侧：排序按钮（排序：直播总销量 / 粉丝数 / 累计观看人数 + ↓/↑ + 下拉）
  - 最右：升降序切换按钮（ArrowUpDown）
- 列表：直播封面、直播标题、主播 ID、国家、指标（粉丝数/累计观看人数/直播总销量）

**受理标准**：
- [ ] 排序下拉选项严格为：直播总销量、粉丝数、累计观看人数
- [ ] 点击 ArrowUpDown 可切换升/降序
- [ ] 列表排序与展示字段一致

**优先级**：P0

---

### 10. 评论子页面（全屏弹窗/内置页）

**用户故事**：
作为运营，我想快速浏览评论情绪与观点聚合，以便判断商品口碑与风险。

**功能点**：
- 情绪筛选：全部/正向/负向
- 观点聚合：正向/负向观点切换
- 评论列表：内容、时间、用户信息（MVP 可静态）

**受理标准**：
- [ ] 筛选/切换有明显选中态
- [ ] 列表可滚动

**优先级**：P1

---

### 11. SKU 详情子页面（弹窗）

**用户故事**：
作为运营，我想在详情页快速查看 SKU 明细，以便对比不同规格表现。

**功能点**：
- 从商品信息卡进入
- 展示 SKU 维度列表（如颜色/尺码）与指标

**受理标准**：
- [ ] 弹窗可打开/关闭

**优先级**：P1

---

### 12. 分享/导出（通用能力）

**用户故事**：
作为运营，我希望把当前页面数据分享给同事或导出线索，以便协作。

**功能点**：
- 分享：复制链接/生成海报（MVP 可先 UI）
- 导出：填写邮箱并提示导出任务提交（MVP 可先 UI）

**受理标准**：
- [ ] 分享/导出入口可见且可触发弹窗
- [ ] 弹窗表单有基本校验（如邮箱格式）

**优先级**：P1

## 非功能需求

### 性能
- 首屏可交互时间（H5）：在中端机/4G 环境下 ≤ 3s（MVP 可先不严格，但需监控）
- 列表滚动应保持流畅，无明显掉帧（≥ 45fps 体感）

### 易用性
- 关键交互控件（筛选/排序/Tab/返回）必须有明确的选中态与点击反馈
- 全屏子页面返回路径清晰（左上返回）

### 可靠性
- 页面异常时给出兜底提示（MVP 可先 toast/空状态）

### 安全
- 不在前端硬编码敏感 Key

## 范围外（本 PRD 不包含）
- 实际后端接口与数据埋点方案的落地细节（可另文档补充）
- 登录/会员购买的完整支付闭环
- 多语言/国际化完整策略
